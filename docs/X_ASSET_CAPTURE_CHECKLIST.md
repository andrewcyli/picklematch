# PickleballMatch.fun — X Asset Capture Checklist

## Goal
Create the minimum visual asset pack needed to start posting on X with strong product clarity.

Priority assets:
1. **QR join screen recording**
2. **Leaderboard / match rail screenshot**
3. **Clean product screenshot**
4. **Optional banner image for X profile**

---

## Asset 1 — QR join screen recording

### Purpose
Used in:
- pinned thread tweet 4
- launch post 2

### What it needs to show
Capture this sequence clearly:
1. organizer opens session
2. QR code is visible
3. player scans or joins via QR flow
4. player appears in session
5. no login / no install friction is obvious

### Ideal duration
- **8–15 seconds**
- short enough to loop

### Capture notes
- keep cursor/taps deliberate and slow enough to follow
- avoid visual clutter
- crop tightly around the flow
- if possible, record on mobile viewport or show mobile-first experience
- make sure session/player names look clean and intentional

### Suggested caption/overlay
- “Scan QR → join session instantly”
- “No app install. No account setup.”

### Quality bar
- QR code readable
- join result obvious
- no awkward pauses
- no broken or placeholder states on screen

---

## Asset 2 — Leaderboard / match rail screenshot

### Purpose
Used in:
- pinned thread tweet 5
- launch post 4
- proof/demo posts

### What it needs to show
- active session state
- visible leaderboard or match rail
- obvious sense that the session is live and structured
- ideally enough players/courts to feel real, but not crowded

### Best composition
- leaderboard centered and readable
- top few players visible
- surrounding UI present enough to show product context
- use realistic names, avoid junk test data

### Capture notes
- prefer desktop or tablet-sized clean layout if it reads better
- avoid including browser chrome if possible
- take 2–3 variations:
  - leaderboard-focused
  - match-rail-focused
  - wider session view

### Suggested caption/overlay
- “Live leaderboard during play”
- “Everyone can see what’s happening”

---

## Asset 3 — Clean product screenshot

### Purpose
Used in:
- launch post 7
- general product announcement posts
- DMs / profile references

### Best candidates
Choose one of:
- session setup screen
- active player queue view
- session overview with clean names and structure

### Selection criteria
Pick the screen that best communicates:
- this is easy to use
- this is for organizing pickleball
- this works right now

### Capture notes
- keep names realistic and friendly
- use a clean, polished session state
- avoid empty-state UI for this asset
- no dev-looking test values

### Suggested caption/overlay
- “Free to use. Works on any device.”
- “Run your pickleball night without the chaos.”

## Content polish checklist
Before capturing any asset:
- use the current approved logo
- clean up test player names
- avoid unfinished UI states
- avoid empty screens unless intentionally showing setup simplicity
- make sure visible copy says **PickleballMatch.fun**
- confirm no embarrassing dev strings / placeholder text

## Capture workflow
1. prepare one clean demo session
2. prepare realistic player names
3. capture QR join recording
4. capture leaderboard screenshot
5. capture clean product screenshot
6. export/crop for X
7. pair each asset to the planned posts
