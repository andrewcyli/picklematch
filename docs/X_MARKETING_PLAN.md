# PickleballMatch.fun — Initial X Marketing Plan

## Goal
Get the first wave of pickleball **organizers/hosts** to discover, try, and reuse PickleballMatch.fun for round robin nights.

## Core positioning
**The easiest way to run a pickleball round robin night.**

Lead with:
- free
- no login
- no download
- QR code join
- live scoring
- live leaderboard

Do **not** lead with:
- generic sports-tech language
- enterprise/tournament software framing
- feature dumping

## ICPs to target first
1. **Round robin hosts**
   - weekly club/community/church/office/condo organizers
   - pain: admin chaos, score confusion, player rotation friction

2. **Club / venue operators**
   - need something low-friction and visible for players

3. **Community builders / coaches**
   - run casual events and want smooth logistics

4. **The default organizer friend**
   - always ends up managing the night manually

## Brand voice for X
- useful
- social-native
- confident but not loud
- community-first
- slightly cheeky, never cringe

### Voice rules
- talk like an organizer, not a pitch deck
- pain first, then product relief
- short, visual phrases: "scan, join, play" / "live leaderboard" / "no login"
- clear > clever

## Best messaging lines
### Taglines
- No login. No download. Just play.
- Run your pickleball round robin without the chaos.
- Live scoring for round robins, minus the clipboard.
- Less admin. More pickleball.

### Recommended X bio
**Free web app for pickleball round robins. Live scoring, live leaderboards, QR join. No login.**

## Content pillars
1. **Organizer pain → product fix**
   - score tracking chaos
   - "who's next?"
   - group chat + screenshot mess

2. **Demo / product proof**
   - QR join clips
   - live leaderboard screenshots
   - match flow screen recordings

3. **Operator tips**
   - how to run smoother round robins
   - reduce downtime on 1–2 courts
   - keep casual players engaged

4. **Social proof**
   - usage screenshots
   - testimonials
   - case studies from real hosts

## Launch sequence
### Phase 1 — prep
Before posting heavily:
- finalize X bio
- create pinned demo post
- prepare:
  - 3 short screen recordings
  - 5 clean screenshots
  - 1 "how it works" image
- use CTA:
  - "Reply 'demo' and I'll send the link"
  - or direct "Try it free for your next round robin"

### Phase 2 — soft launch (week 1)
- 1–2 posts/day
- reply to relevant pickleball organizer conversations
- manually DM a handful of organizers if relevant
- goal: first usage, first screenshots, first testimonials

### Phase 3 — proof (week 2)
- turn early usage into posts:
  - testimonial posts
  - before/after workflow posts
  - mini case studies
- push the frictionless angle harder once proof exists
