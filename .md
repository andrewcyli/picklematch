# PickleMatch Restart: Product Brief

## Vision
A streamlined, social-focused round-robin pickleball scheduler designed for casual group nights. Two courts max, smart AI rotation, zero tournament complexity.

---

## Core User Journeys

### 1. Host Sets Up a Session (30 seconds)
1. Arrives at `/start` → taps "Create Session"
2. Chooses: **2 courts** (locked), game length (15/20/25 min), total session time (1.5h/2h/2.5h)
3. Optional: Mark Court A as singles, Court B as doubles (default: both doubles)
4. System generates 6-character game code (e.g., `X7K2P9`)
5. Host shares code via WhatsApp/iMessage

### 2. Players Join (10 seconds each)
1. Player opens link or visits `/join`
2. Enters 6-character code
3. Types their name (or selects from recent players list)
4. Lands on "My Matches" view showing personal schedule

### 3. Running the Session (Ongoing)
1. Host sees **Live Board**: Now/Next for both courts, bench list, queue preview
2. Players see **My Matches** + current court status
3. Score entry happens on the Live Board (host) or via simple modal (players)
4. AI auto-rotates players after each game; bench auto-updates
5. Late arrivals tap "Join" → enter name → instantly slotted into future rounds
6. Early departures tap "Leave" → removed from future matches, current matches preserved

### 4. Session Wrap (5 minutes)
1. Host taps "Wrap Up" or session auto-ends at time limit
2. Final leaderboard displays with win rates, point differentials, matches played
3. "Share Night's Stats" generates image for WhatsApp/Instagram Stories
4. Option to export CSV for spreadsheet nerds

## Must-Keep Logic (From Existing QuickCourt + Classic)

### AI Rotation Engine (`src/lib/scheduler.ts`)
- **Smart pairing algorithm**: Prioritizes players who've played least, avoids back-to-backs, respects rest periods (2+ consecutive = forced rest)
- **Partner diversity**: Tracks recent partners/opponents, favors new combinations
- **Court balancing**: Distributes play across both courts evenly
- **Locked pairs support**: Honors preset teammate pairs when configured
- **Matchup tracking**: `MatchupTracker` class prevents repeat opponents where possible
- **Regeneration on player change**: Preserves completed/current matches, regenerates future slots only

### Data Model (`src/core/types/index.ts`)

### Key Features
- **Check-in/out**: Players can join/leave mid-session; schedule regenerates intelligently
- **Score entry**: Simple numeric inputs, validates winners, updates leaderboard in real-time
- **Player identity**: "Find my matches" lets players view personal schedule without seeing full host view
- **Live timers**: Per-court elapsed time tracking
- **Skip next**: Player can opt to sit out one round (5-min auto-reset)
- **Supabase sync**: Real-time updates across all connected devices

## Data/State: Players Joining/Leaving Mid-Session

### Current Behavior (Preserve)
When players change after matches have started:

1. **Preserve completed matches**: Scores locked, contribute to leaderboard
2. **Preserve current matches**: Active games continue with original players
3. **Regenerate future slots only**: From `regenerateFromTime = max(completedMatch.endTimes)`
4. **Sanitize match IDs**: Prevent duplicates during regeneration
5. **Sync to Supabase**: All clients see updated schedule instantly

### Edge Cases to Handle
| Scenario | Behavior |
|----------|----------|
| Player leaves mid-match | Finish current game, remove from future slots |
| Player leaves while on bench | Immediate removal, no schedule impact |
| New player joins during active session | Slotted into future rounds based on fairness algorithm |
| Drop to <4 players | Show "Need 4+ players" message, pause new match generation |
| All players leave | Keep session alive for 24h, allow rejoin with same code |

### State Management
```typescript
// Core state (per session)
interface SessionState {
  gameCode: string;
  players: string[];              // Current roster
  matches: Match[];               // All matches (completed + current + future)

}
```

## Score Entry Needs

### Host Flow (Live Board)
- Inline score inputs on Now/Next cards
- Validation: Prevent ties, max score 15 (win by 2) or 11 (win by 2) based on config
- Auto-advance: After score entry, next match auto-promotes to "Now"
- Undo: Allow score correction within 30 seconds (with toast confirmation)

### Player Flow (My Matches)
- Simple modal: "Enter score: Us [__] vs Them [__]"
- Submit goes to host for approval (prevents disputes)
- Host sees pending score badge, taps to confirm/override

### Stats Calculated
- Wins/Losses per player
- Win rate %
- Total points scored/allowed
- Point differential (+/-)
- Matches played count

## Social Stats / Leaderboard / Share Moments

### Live Leaderboard (During Session)
- Top 5 preview on Live Board
- Sort: Win rate → Point differential → Matches played
- Visual flair: Crown icon for #1, trend arrows (▲▼) vs previous position

### Final Wrap View
- Full leaderboard with all stats
- "Night MVP" highlight for top performer

   - Session title: "Friday Night Pickleball"

### Export Options

## Recommended Ad Slots (Future AdSense)

### Tier 1: High Visibility, Low UX Impact
| Slot | Placement | Format | Trigger |
|------|-----------|--------|---------|
| **A1** | Below Live Board, above "Detailed Controls" fold | 728x90 (desktop) / 320x50 (mobile) | Always visible on scroll |
| **A2** | Wrap view, below leaderboard, above match history | 336x280 (large rectangle) | Session ended only |
| **A3** | "My Matches" empty state | Native-style text ad | When player has 0 upcoming matches |

### Tier 2: Medium Visibility
| Slot | Placement | Format | Trigger |
|------|-----------|--------|---------|
| **B1** | Player setup screen, below roster list | 300x250 (medium rectangle) | 4+ players added |
| **B2** | Between "Now" and "Next" cards (subtle) | 320x50 (mobile banner) | After 3+ scores entered |

### Tier 3: Minimal/Experimental
| Slot | Placement | Format | Notes |
|------|-----------|--------|-------|
| **C1** | Post-score interstitial | Full-screen (time-limited) | Show every 5th score entry, 5s max, skippable |
| **C2** | Session recap share page | Native | Only on shared/view-only pages |

### UX Guidelines for Ads
- **Never** place ads over interactive elements (score inputs, buttons)
- **Never** insert ads mid-scroll of match lists (causes layout shift)
- **Always** reserve space to prevent CLS (Cumulative Layout Shift)
- **Consider** dark mode compatibility for ad backgrounds
- **Respect** `prefers-reduced-motion` for animated ad content
